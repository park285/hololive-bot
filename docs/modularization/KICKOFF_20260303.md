## 1) 사전 검증 결과

### Rust (hololive-rs)

- `crates/shared/services/{cache,configsub,member,ratelimit,parser}` 5개 crate는
  - workspace 등록은 되어 있으나
  - crate 본인 `Cargo.toml` 외 소비자 참조가 확인되지 않음
- `shared-{formatter,notification,template}` 3개 crate는
  - `crates/dispatcher/app/Cargo.toml`에서만 소비 확인

결론:
- Phase 0(5개 삭제) -> Phase 1(3개 dispatcher 하위 이동) 순서가 유효함

### Go (hololive-shared)

- `pkg/health`는 admin/bot/llm-sched/stream-ingester에서 공통 사용 확인
- `pkg/repository`는 admin + bot에서 사용 확인

결론:
- `health`, `repository`를 admin 내부로 이동하는 기존 가정은 철회
- 두 패키지는 shared 유지가 맞음 (Phase 4 문서 반영 필요)
